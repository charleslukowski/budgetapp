# Monthly Input Patterns

## The Challenge

Some drivers need monthly granularity (coal prices, use factors, deliveries), but showing 12 individual inputs is:
- Overwhelming
- Tedious to enter
- Hard to spot patterns or outliers

## Design Patterns for Monthly Inputs

---

## Pattern 1: "Annual with Monthly Override"

Best for: **Coal Prices** - Usually flat or with known escalation, occasional month-specific overrides

## Recommended Combinations

| Driver Type | Primary Pattern | Secondary Pattern |
|-------------|-----------------|-------------------|
| **Coal Prices** | Annual + Override (#1) | Roll Forward Variance (#6) |
| **Use Factors** | Visual Curve (#2) | Annual + Override (#1) |
| **Barge Rates** | Sparkline + Expand (#4) | Seasonal presets |
| **Contract Deliveries** | Spreadsheet Grid (#3) | Paste from Excel |
| **Heat Rates** | Single annual value | Monthly only if cycling |
| **Reagent Prices** | Annual only | Rarely need monthly |

## Summary

For monthly coal prices specifically:

1. **Default to annual** - Most months are the same price
2. **Offer patterns** - Flat, quarterly step-up, seasonal
3. **Allow overrides** - Click specific month to change
