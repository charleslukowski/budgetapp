# Fuel Model Workflow - Expanded Concept 3

## Core Principles

1. **Monthly calculations are the foundation** - Everything computes monthly, summarizes to annual for display
2. **Roll forward by default** - Start from last month's forecast, update only what changed
3. **Monthly update cadence** - Designed for at-most-monthly updates
4. **Preserve what works** - Carry forward stable inputs, highlight what needs attention
5. **Track changes** - Clear visibility into what changed from prior forecast

## Summary

This workflow is designed for **monthly rolling forecasts** where:

1. **Most inputs stay stable** - Coal prices, heat rates, reagents don't change often
2. **Position data updates** - Inventory levels, actuals replace forecasts
3. **Exceptions are highlighted** - Outage changes, price moves get flagged
4. **Comparison is built-in** - Always see variance from prior forecast
5. **Notes capture context** - Document why things changed

The "Keep Prior" default for stable items means a typical monthly update might only require updating inventory positions and confirming everything else - a 5-minute process rather than rebuilding from scratch.
